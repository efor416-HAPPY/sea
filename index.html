네이버 서치어드바이저 제휴 API 연동 (Crawl Request)

연동 오류 방지 및 에러 해결을 위한 필수 체크리스트
  • 소유확인된 도메인과 완벽한 일치 여부 (가장 흔한 실수)
    서치어드바이저 웹마스터 도구에 소유 확인을 완료한 대표 사이트 주소와 API 요청 URL이 스키마(http/https) 및 www 유무까지 완벽하게 일치해야 합니다.
    - 올바른 예: https://your-site.com/subpage (등록 주소가 https://your-site.com인 경우)
    - 잘못된 예: https://www.your-site.com/subpage (www 누락/추가 시 다른 사이트로 인식)
  • Access Token 만료 및 오타/공백 확인
    발급받은 토큰이 유효한지 확인하고 복사/붙여넣기 시 공백이나 누락된 문자가 없어야 합니다. 필요시 웹마스터 도구 설정에서 토큰을 재발급받아 적용해 보세요.
  • 리다이렉트(Redirect) 유무
    수집을 요청하는 URL은 브라우저에서 최종적으로 열리는 실제 랜딩 페이지여야 합니다. 해당 URL이 다른 페이지로 강제 이동(Redirect)되도록 설정되어 있으면 수집 요청이 실패로 처리됩니다.
